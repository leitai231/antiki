# 不想背单词 Phase 0/1 验收清单

更新日期：2026-03-13

说明：以下勾选基于“代码实现 + 本地构建通过”口径；浏览器 URL 功能已延后，当前版本不做。

## 验收前准备

- [x] 使用 Debug 构建并运行应用。
- [ ] 系统已授权通知权限。
- [ ] Safari / Chrome / Edge / Arc / Firefox 至少准备其中两个浏览器用于来源测试。
- [ ] 准备 3 条测试文本（英文句子、含标点句子、较长句子）。

## Phase 0 验收

### P0-1 全局快捷键触发

- [x] 在任意非 不想背单词 App（如 Notes）中复制一句话。
- [x] 按 `Cmd+Shift+D` 后弹出“选择生词”面板。
- [x] 关闭后再次按热键可再次弹出。

通过标准：触发稳定，连续 5 次无卡死或无响应。

### P0-2 剪贴板读取 + 分词展示

- [x] 面板中显示原句。
- [x] 下方 token 区域显示可点击词和标点。
- [x] 纯空白剪贴板时出现“剪贴板为空”提示。
- [x] 超长文本（>1000 字符）时出现“文字太长”提示。

通过标准：文本读取与分词符合预期，异常输入有明确提示。

### P0-3 来源 App 识别

- [x] 在非浏览器 App 触发时，面板下方显示正确 App 名称。
- [x] 在浏览器触发时，面板下方显示正确 App 名称。

通过标准：来源 App 名称与触发前前台 App 一致。

### P0-4 弹窗可关闭性

- [x] 点击右上角关闭按钮，面板立即关闭。
- [x] 点击“取消”按钮，面板立即关闭。
- [x] 关闭后状态恢复正常，可重新触发。

通过标准：不存在“弹窗无法关闭”或“关闭后无法再次打开”的情况。

## Phase 1 验收

### P1-1 选词交互

- [x] 点击词可选中/取消选中。
- [x] 支持多选，已选区域显示正确。
- [x] `Enter` 可确认添加，`Esc` 可取消。

通过标准：交互无明显延迟，键盘行为符合预期。

### P1-2 落库与状态流转

- [x] 确认添加后，主列表出现新记录。
- [x] 状态可见 `pending -> processing -> completed`。
- [x] 失败任务可点击“重试”。

通过标准：每次采集都有对应记录，状态更新无丢失。

### P1-3 浏览器 URL 获取（已延后）

- [ ] 本项延后到后续阶段，当前版本不做验收。

通过标准：N/A（当前版本只记录来源 App，不抓 URL）。
